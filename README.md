## рҹ“ҳ About LIU

LIU is a symbolic language that uses emojis combined with logical operators
to create structured meaning.

Unlike casual emoji usage, LIU defines:

- grammar rules
- semantic modifiers
- compositional logic

The goal is to create a **universal intuitive language** readable across cultures.

---

## вҡҷпёҸ How LIU Works

Each emoji has a **general meaning**, not just its visual meaning.

Example:

- вҳҖпёҸ вҶ’ day  
- *вҳҖпёҸ вҶ’ star  
- *вҳҖпёҸрҹҢҺ вҶ’ Sun  

Meaning is expanded using operators and combinations.

## рҹ§  Core Concepts

### 1. Semantic Modifiers

- `*` вҶ’ object / noun  
- `#` вҶ’ quality / adjective  
- `!` вҶ’ action / verb  

Example:

- вқӨпёҸ вҶ’ love  
- *вқӨпёҸ вҶ’ heart  
- !вқӨпёҸ! вҶ’ to love  
- #вқӨпёҸ вҶ’ loving  

---

### 2. Emoji Combination

Meaning is built by combining symbols:

- рҹҸ  вҶ’ building  
- *рҹҸ  вҶ’ house  
- рҹҸ рҹҷӮ вҶ’ personal residence  
- рҹ§ҚвҖҚвҷӮпёҸ*рҹҸ  вҶ’ resident

- `@` вҶ’ identity / name

### 5. Structure & Logic

LIU uses operators for:

- hierarchy (`^`, `v`)
- comparison (`<`, `>`)
- time (`|`, `/`)
- grouping (`(`, `[`, `{`)

### 5.1. Hierarchy

- `^` вҶ’ above
- `v` вҶ’ below
- `=` вҶ’ same level

Examples:

- `^рҹҷӮ` вҶ’ father
- `=рҹҷӮ` вҶ’ you
- `vрҹҷӮ` вҶ’ son

### 5.2. Comparison

- `<` вҶ’ smaller
- `>` вҶ’ larger

Examples:

- `<*рҹҢІ` вҶ’ bonsai
- `>*рҹҢІ` вҶ’ sequoia

### 5.3. Time

- `|` вҶ’ beginning
- `/` вҶ’ end

Examples:

- `|рҹҷӮ` вҶ’ birth
- `/рҹҷӮ` вҶ’ death

### 5.4. Composition and scope

- `]` вҶ’ what something is made of
- `)` вҶ’ more fundamental component or content
- `}` вҶ’ broader concept

Examples:

- `]*рҹҢІ` вҶ’ wood
- `)*рҹҢІ` вҶ’ cellulose
- `}рҹҢІ` вҶ’ vegetation
- `]*рҹҸ ` вҶ’ brick
- `)*рҹҸ ` вҶ’ mud
- `}*рҹҸ ` вҶ’ residential

### 5.5. Grouping

- `(` вҶ’ small group
- `[` вҶ’ larger group
- `{` вҶ’ supergroup

Examples:

- `(*рҹҢІ` вҶ’ grove
- `[*рҹҢІ` вҶ’ forest
- `(рҹҸ ` вҶ’ neighborhood
- `[рҹҸ ` вҶ’ city
- `{рҹҸ ` вҶ’ state
- `вҳҖпёҸ` вҶ’ day
- `(вҳҖпёҸ` вҶ’ week
- `[вҳҖпёҸ` вҶ’ month
- `{вҳҖпёҸ` вҶ’ year

## 6. LIU Endings

LIU also uses endings to mark gender, number, verb tense, and possession.

### Gender and number

- `x` вҶ’ neutral
- `a` вҶ’ feminine
- `o` вҶ’ masculine
- `:` вҶ’ plural

Examples:

- `рҹҷӮx` вҶ’ neutral person
- `рҹҷӮa` вҶ’ woman
- `рҹҷӮo` вҶ’ man
- `рҹҷӮ:` вҶ’ people

## 7. Possession

Possession is indicated with `&`:

- `&рҹҷӮ` вҶ’ my / mine
- `&рҹҷӮ:` вҶ’ my / mine (plural)
- `&=рҹҷӮ` вҶ’ your / yours
- `&=рҹҷӮx` вҶ’ your / yours
- `&=рҹҷӮa` вҶ’ hers
- `&=рҹҷӮo` вҶ’ his

## 8. Grammatical links and interrogatives

LIU has short and regular links to build sentences.

Examples:

- `_1` вҶ’ one
- `_x` вҶ’ of
- `_>x` вҶ’ in
- `_рҹҷӮo` вҶ’ this
- `_&o` вҶ’ of this
- `_>рҹҷӮo` вҶ’ that
- `_рҹҸ ` вҶ’ here
- `_>рҹҸ ` вҶ’ there
- `_)` вҶ’ with
- `_>` вҶ’ for
- `_рҹҷӮ?` вҶ’ who
- `_рҹҸ ?` вҶ’ where
- `_вҸұпёҸ?` вҶ’ when
- `_@?` вҶ’ which
- `_$?` вҶ’ how much

## 9. Use of aliases

LIU allows **aliases** between common emojis, which are not part of the list of basic emojis,
and combinations formed by two or more basic emojis.

This makes it possible to use emojis that already exist on the keyboard or mobile phone
without needing to create a new word in every language. Instead, the system treats certain
common emojis as equivalent to compositions already defined in LIU.

Examples of compositions with basic emojis:

- `рҹ‘•рҹҰ¶рҹҸ»` вҶ’ shoe
- `рҹ‘•рҹҰ¶рҹҸ»a` вҶ’ womenвҖҷs shoe
- `рҹҷӮрҹ‘•рҹҰ¶рҹҸ»` вҶ’ sneaker
- `%рҹ‘•рҹҰ¶рҹҸ»` вҶ’ boot
- `*рҹҢІ` вҶ’ tree
- `*рҹҢІ~рҹҚҖ` вҶ’ cactus

Examples of aliases:

- `рҹ‘һ вҶ” рҹ‘•рҹҰ¶рҹҸ»`
- `рҹ‘  вҶ” рҹ‘•рҹҰ¶рҹҸ»a`
- `рҹ‘ҹ вҶ” рҹҷӮрҹ‘•рҹҰ¶рҹҸ»`
- `рҹҘҫ вҶ” %рҹ‘•рҹҰ¶рҹҸ»`
- `рҹҢі вҶ” *рҹҢІ`
- `рҹҢө вҶ” *рҹҢІ~рҹҚҖ`

In the case of an alias such as `рҹҢі вҶ” *рҹҢІ`, there is a direct one-to-one correspondence.
This is important because many emojis have very similar drawings. By defining aliases
of this kind, LIU starts treating these variations as equivalent within the system.

Since aliases relate **emoji to emoji**, this definition applies equally to
all languages. Thus, it is defined only once and can be reused throughout LIU.

In this way, the language remains organized around a reduced set of basic emojis,
without losing the practical flexibility of also using the many other existing emojis.

## 11. Objective and importance of LIU

Because it is a pictographic language, based on symbols and not on words from a single language,
LIU can be read by users of different languages in their own local translation.

For example:

- `рҹҷӮ !вқӨпёҸ =рҹҷӮ`

it can be read as:

- **Eu amo vocГӘ** вҶ’ for Portuguese speakers
- **I love you** вҶ’ for English speakers
- **Ich liebe dich** вҶ’ for German speakers

Thus, LIU has the potential to facilitate communication between people of different languages.

In addition, LIU can be useful in:

- initial adult literacy
- children's learning
- intuitive teaching of verb, adjective, and noun
- multilingual signage
- playful use by children and young people
- philosophical reflection on the meaning of words

For example:

- `рҹҷӮ` вҶ’ I
- `вқӨпёҸ` вҶ’ love
- `=рҹҷӮ` вҶ’ you
- `вқӨпёҸрҹҷӮa` вҶ’ girlfriend
- `>вқӨпёҸрҹҷӮa` вҶ’ wife
- `(вқӨпёҸрҹҷӮa` вҶ’ family

In these cases, besides generating the desired word, LIU also makes explicit
a structure of meaning.

This kind of development would hardly be carried out by a single person without computational support.
Artificial intelligence made it possible to take this concept from paper and implement it in practice.

Anyone who wishes to help in the development of LIU, through technical collaboration,

## рҹ—ӮпёҸ How to Improve the LIU Language Databases

Anyone who wants to help can improve the LIU language databases.

Currently, the website www.liu777.org uses the Python program liu.py to load its databases, which currently contain emoji and text data in JSON format, such as:
- `LIUv1-pt.json`
- `TXT_LIUv1-pt.json`
- `LIUv1-ig.json`
- `TXT_LIUv1-ig.json`

These are databases in Portuguese and English created from Python programs:
-`dbliu_pt.py` вҶ’ LIU database in Portuguese
-`dbliu_ig.py` вҶ’ LIU database in English

For other languages, this repository contains only the basic LUI framework for each language:
- `dbliu_fr.py` вҶ’ LIU database in French
- `dbliu_ru.py` вҶ’ LIU database in Russian
- `dbliu_es.py` вҶ’ LIU database in Spanish
- `dbliu_it.py` вҶ’ LIU database in Italian
- `dbliu_jp.py` вҶ’ LIU database in Japanese
- `dbliu_ch.py` вҶ’ LIU database in Chinese
- `dbliu_ge.py` вҶ’ LIU database in German

Each language file contains entries with 3 columns, for example:

`["вқӨпёҸрҹҷӮ","couple (sweethearts)","[love + me = couple] @Poli"],`

Where:

- `"вқӨпёҸрҹҷӮ"` вҶ’ LIU symbol
- `"couple (sweethearts)"` вҶ’ main word with possible synonyms
- `"[love + me = couple] @Poli"` вҶ’ explanation of the symbol and authorship

### Notes for contributors
- New words should be placed close to emojis from the same semantic group
- Anyone who creates new words may include authorship, for example: `@Author_name`
- Entries marked with `@Google` or `@GPT5` can be revised and improved
- Future language files should keep the same standard naming pattern for consistency

### Grammatical keys in the third column

Some entries use grammatical keys in the third column to indicate which form
of the word should be used in each grammatical case.

The rule is:

- each expression between `()` in the **second column**
- corresponds to one grammatical key in the **third column**
- in the same order

Example in Portuguese:

`["!рҹҷӮ","Г© (sГЈo) (sou) (Г©s) (Г©) (somos) (sois) (sГЈo)","(num_plu) (1ps) (2ps) (3ps) (1pp) (2pp) (3pp) @Poli"],`

This means:

- `Г©` вҶ’ default form
- `(sГЈo)` вҶ’ plural form вҶ’ `(num_plu)`
- `(sou)` вҶ’ first person singular вҶ’ `(1ps)`
- `(Г©s)` вҶ’ second person singular вҶ’ `(2ps)`
- `(Г©)` вҶ’ third person singular вҶ’ `(3ps)`
- `(somos)` вҶ’ first person plural вҶ’ `(1pp)`
- `(sois)` вҶ’ second person plural вҶ’ `(2pp)`
- `(sГЈo)` вҶ’ third person plural вҶ’ `(3pp)`

Another examples:

`["!рҹҷӮ:","sГЈo (somos) (sois) (sГЈo)","(1pp) (2pp) (3pp) @Poli"],`

`["!рҹҷӮv","foi (fui) (foste) (foi) (fomos) (fostes) (foram)","(1ps) (2ps) (3ps) (1pp) (2pp) (3pp) @Poli"],`

`["!рҹҷӮ^","serГЎ (serei) (serГЎs) (serГЎ) (seremos) (sereis) (serГЈo)","(1ps) (2ps) (3ps) (1pp) (2pp) (3pp) @Poli"],`

### Meaning of the grammatical keys

- `(1ps)` вҶ’ first person singular
- `(2ps)` вҶ’ second person singular
- `(3ps)` вҶ’ third person singular
- `(1pp)` вҶ’ first person plural
- `(2pp)` вҶ’ second person plural
- `(3pp)` вҶ’ third person plural
- `(num_plu)` вҶ’ plural form
- `{gen_fem}` вҶ’ feminine gender
- `{gen_masc}` вҶ’ masculine gender

### Example in English

In English, the same logic applies, but usually uses fewer conjugation forms than Portuguese.

For that reason, only the forms that are really needed should be included.

A simple example is:

`["!рҹҷӮ","is (are)","(num_plu) @Poli"],`

This means:

- `is` вҶ’ singular form
- `(are)` вҶ’ plural form вҶ’ `(num_plu)`

So, if a language does not use a certain distinction, that key should simply be omitted.
